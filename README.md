# Автоматическая разметка сборника статей для НЭБ/РИНЦ

## 🎯 Предназначение

Данный репозиторий содержит код для автоматической XML-разметки сборника статей для последующего размещения в НЭБ/РИНЦ.
Предназначен и использовался для составления сборников трудов конференции [АПНХ](https://www.apinch.ru). После кастомизации может быть полезен для аналогичных задач (парсинга word- и pdf-файлов и составлении xml-разметки на их основе).
Парсинг базируется на регулярных выражениях и строгой структуре каждой статьи. Производительность скриптов засивист от чёткости верстки и соблюдении шаблонов изначальных файлов – ТРЕБУЕТ РУЧНОЙ ПЕРЕПРОВЕРКИ!!!!

## ⚙️ Использование

Необходимо указать путь для .DOCX (по нему парсится содержимое) и до .PDF (по нему идёт определение страниц каждой статьи), а также локальных переменных.
Лучше запускасть full_stack.ipynb – автоматически отработает. Можно залить ноутбук на [Colab](https://colab.research.google.com), чтобы не запускать код локально.
Можно расширить списки Базовых организаций (чтобы отделять от структурных подразделений) и Триггеры на абзацы с финансированием.
Остальные детали постарался подробно описть в комметариях, docstrings и аннотациях.

#### 🧑🏼‍🦯‍➡️Последовательность действий

- Закинуть в корневую папку файлы .docx и .pdf сборника (предворительно убедиться, что верстка в них совпадает!!!), прописать их названия в соответсвующих переменных или полный путь к ним в ином случае. При работе в Colab – закинуть эти файлы в корень проекта;
- Тоже самое сделать для файла обложки;
- Изменить переменные с мета-информацией;
- Создать итератор с названиями фалов каждой статьи или воспользоваться готовой функцией получения их названий, если файлы уже подготовлены;
- Запустить код;
- Можно дополнить списки "BASE_ORGS" (список организаций для отделения структурных подразделений) и "TRIGGERS" (список выражений для определения абзаца с финансированием) для улучшения качества автоматической разметки;
- Выполнить код, контролируя промежуточные выводы;
- > Если парсер падает из-за большого объма файла – удалить картинки обложек и перезапустить!!!
- Можно кастомить код для улучшения разметки и подгона под изменения шаблонов;
- Вручную проверифицировать содержание всех тегов!!!!!!!!!
- Проверить валидность итогового .xml после ручной правки;
- Сформировать архив, содержащий .xml итоговой разметки, .pdf всего сборника, .jpeg обложки, .pdf-файлы каждой статьи, отправить ответсвенному для публикации в библиотеке.
